# WooCS.ai — WordPress Plugin Development Rules

## 1. Project Structure
- All plugin code lives in `./plugin/`.
- Main plugin file: `plugin/woocs-ai.php`.
- The `plugin/` directory is bind-mounted into the WP container — changes reflect instantly, no restart needed.
- Built widget bundle is served from `plugin/assets/woocs-widget.js` (generated by `make wp-build`).

## 2. Architecture
- Plugin communicates with Django backend via authenticated HTTP (`wp_remote_post`, `wp_remote_get`).
- All requests to Django carry `X-API-Key` header (stored in `wp_options`).
- Widget config is injected via `wp_localize_script` as `window.WooCS = {store_id, api_url, store_name}`.
- Admin pages registered under the `woocs-settings` menu slug.

## 3. Admin Pages
| Page | Slug | Purpose |
|---|---|---|
| Settings | `woocs-settings` | WC credentials, API key, widget toggle |
| Sync | `woocs-sync` | Sync status, manual sync trigger |
| FAQs | `woocs-faqs` | FAQ CRUD, sync FAQs |
| Preview | `woocs-preview` | Iframe storefront preview with debug overlay |

## 4. WordPress Standards
- Use `add_action` / `add_filter` exclusively. Never modify core or WC files.
- Use `wp_options` for plugin settings (`get_option`, `update_option`).
- Sanitize all inputs: `sanitize_text_field`, `sanitize_url`, `sanitize_email`.
- Escape all outputs: `esc_html`, `esc_attr`, `esc_url`.
- Use `wp_nonce_field` / `check_admin_referer` on all forms and AJAX actions.
- Capability check: admin pages require `manage_woocommerce` capability.

## 5. Custom Tables
- FAQs stored in a custom WP table: `{$wpdb->prefix}woocs_faqs` — columns: `id`, `question`, `answer`, `updated_at`.
- No other custom tables in PoC.

## 6. Widget Injection
- Enqueue `plugin/assets/woocs-widget.js` via `wp_enqueue_script` on all public-facing pages.
- Exclude WP Admin pages and the order confirmation page.
- Use `wp_localize_script` to inject config before the script.

## 7. Code Standards
- PHP 8.1+. Strict types (`declare(strict_types=1)`).
- OOP: all logic in classes, autoloaded via PSR-4 or explicit `require_once` (Composer not required for PoC).
- All class/function/variable names and comments in English.

## 8. Security
- Encrypt stored WC consumer key and secret at rest using `wp_hash` or a reversible encryption utility.
- API key never exposed to the browser.
- All AJAX endpoints registered with `wp_ajax_` hooks and nonce-verified.
